1. 简介

欢迎使用 Double Commander！

1.1. 起源

Double Commander – 是一个由俄罗斯程序员 Alexander Koblov (Alexx2000@mail.ru) 开发的项目，作为 Sexi Commander (作者 – Radek Cervinka, radek.cervinka@centrum.cz) 的一个分支。

（返回顶部）

1.2. 使用

获取 Double Commander 有几种方法：

  • 访问 GitHub 上的 发布 页面。

  • 访问 SourceForge 上的 发布 页面。

  • Double Commander 可在许多 Linux 发行版的仓库和 FreeBSD Ports 集合中找到。

  • Double Commander 为多个 Linux 发行版提供了自己的 仓库（Linux 发行版的官方仓库并不总是包含程序的当前（最新）版本）。

  • 您还可以自己 构建 Double Commander。

版本存档 和 夜间构建 也可用。

首先，您必须选择操作系统（Windows、Linux、macOS）、处理器架构，然后选择要下载的版本。

对于 Windows，有一个可执行文件 .exe（Inno Setup）或 .msi 和一个压缩包 .zip（即所谓的便携版本）。

对于 Linux，有基于 GTK 工具包和基于 Qt（Qt5 或 Qt6）的版本。还提供：便携版本（与 XZ 打包的编译二进制文件）和 AppImage 格式的文件。

在第一次启动时，Double Commander 将自动检测系统语言，并为界面设置该语言。DC 已翻译成多种语言。语言可以在 程序设置 中更改。

Double Commander 确定并使用当前系统主题。它看起来像这样（Ubuntu，三个主题和两个图标集）：

从图中可以看出，左侧是文件面板，底部是命令行，下面的窗口是日志查看器（显示文件管理器的所有操作）。Double Commander 还支持 Windows 的深色模式（Windows 10 1809 及更高版本）。

Double Commander 使用 Total Commander 插件 API，因此在 Windows 下可以使用 Total Commander 的插件（其中许多与 DC 兼容）。有关插件类型的描述，请参见 插件 设置部分。DC 附带了几个 预安装插件，其他插件可以手动安装。

2. 界面

Double Commander 的界面。

2.1. 菜单栏

菜单栏提供对 Double Commander 所有功能的访问。但请不要忘记，大多数操作可以通过其他方式执行：使用热键或工具栏上的按钮。

2.1.1. "文件"

此菜单包含几乎所有可以对文件执行的操作。请考虑所有项目：

创建符号链接...（内部命令 cm_SymLink）。使用此项，Double Commander 会创建一个指向光标下文件的 符号链接，并将其放置在对面面板中打开的文件夹中。

链接名 – 这是链接的路径及其名称。名称是行中最后一个目录分隔符字符后的最后一个单词。

链接将指向的目标 – 链接所指向的对象。

尽可能使用相对路径 – 如果勾选，Double Commander 将使用相对路径指向对象目录。

创建硬链接...（内部命令 cm_HardLink）。与上面的符号链接类似，但链接是 硬链接，并且使用相对路径将不可用。

创建文件夹（内部命令 cm_MakeDir） – 在当前文件夹中创建一个新的子文件夹。光标下的文件名（不带扩展名）或文件夹名将被提供，这对于创建多个具有相似名称的文件夹非常有用。您还可以在一次操作中创建多个子文件夹，只需用目录分隔符分隔子文件夹（在 Unix/Linux 系统中为 "/"，在 Windows 中为 "\"）。

带箭头的按钮打开输入名称的历史记录。输入字段右侧的按钮允许启用或禁用来自目录创建历史的自动完成。

如果启用了 扩展语法，则可以使用 <[] 模版插入当前日期/时间：在方括号内可以使用 日期和时间格式字符 和任何允许用于文件名的字符。

创建快捷方式（仅限 Windows） – Double Commander 会创建一个指向光标下文件的快捷方式（Microsoft Windows .lnk 文件），并将其放置在对面面板中打开的文件夹中。

更改属性...（内部命令 cm_SetFileProperties） – 允许更改时间戳和属性（Windows）或文件权限（Linux 和其他类 Unix 系统），包括子文件夹或不包括子文件夹。

显示文件属性...（内部命令 cm_FileProperties）将显示默认的“属性”对话框（Windows）或 Double Commander 自己的对话框（Linux 和其他类 Unix 系统），如下所示：

指示文件或文件夹名称、MIME 类型、大小、写入和最后访问时间。对于链接，Double Commander 还将显示链接对象（符号链接）或其总数（硬链接）。在“属性”选项卡上，您可以更改文件的访问权限及其所有者（如果您有权这样做）。 “插件”选项卡是可选的，仅在您拥有合适的内容插件时可用（在这种情况下是 DEB）。另请参阅 <系统项属性> 的描述。

使用项目 编辑注释...（内部命令 cm_EditComment），您可以创建或更改文件注释，这些注释存储在文件 descript.ion 中。支持 UTF、ISO、KOI 和 ANSI 编码。在 杂项 设置部分中，您可以设置默认文本编码。

注释仅适用于本地文件系统上的普通文件，而不适用于远程或虚拟文件系统（例如，在 FTP 服务器、压缩包等中）。为了使 Double Commander 在复制或移动文件时考虑 descript.ion 的内容，请确保在 文件操作 设置部分中启用 处理文件/文件夹的注释。

计算占用空间（内部命令 cm_CalculateSpace）项计算所选对象的大小（以字节为单位和四舍五入的单位）。

比较内容 （内部命令 cm_CompareContents） 显示一个带有两个面板的窗口，用于比较文件（通常是文本）。

这是一个简单的视觉比较和合并工具，具有典型的功能：在差异之间导航（下一个、上一个、第一或最后），从右到左和从左到右复制文本块，就地编辑文件，撤消和重做最后的更改，以不同名称保存等。

选项：

  • 自动比较 – 更改 忽略空格 和 忽略大小写 参数时自动进行比较。并且在选择新文件时也是如此。

  • 忽略空格 – 如果启用，则多个空格将被视为一个。

  • 忽略大小写 – 如果启用，则不区分大小写。

  • 差异行 – 如果启用，则还会显示行内的差异。

  • 二进制模式 – 结果以二进制形式呈现。

  • 持续滚动 – 允许同步滚动（垂直和水平）。

使用的颜色可以在 颜色 设置部分中更改（差异 类别）。另请参阅 工具 比较程序。

将选择两个文件，如下所示

  • 如果在活动面板中选择了两个文件，它们将被打开在比较工具中。第一个选定的文件将被打开在比较工具的左侧面板中。

  • 如果只选择了一个文件，它将与非活动面板中选定的文件进行比较。

  • 如果只选择了一个文件，并且在非活动面板中没有选择任何内容，它将与第二面板中具有相同名称的文件进行比较（如果存在），或者与光标下的文件进行比较。

  • 如果没有选择任何内容，光标下的文件将与另一个面板中具有相同名称的文件进行比较（如果存在），或者与光标下的文件进行比较。

在工具 > 比较程序 选项卡中，您可以设置外部比较工具。

批量重命名工具 （内部命令 cm_MultiRename）

允许通过定义的掩码重命名任何文件。例如，一些具有不同名称的文件可以重命名为 Picture1、Picture2、Picture3 等。首先，选择要重命名的文件，然后启动此工具（请参见上图）。在那里，您可以看到文件的旧名称和新名称。有关详细说明，请参阅 此页面。

通过选择项目 压缩文件 （内部命令 cm_PackFiles），我们调用一个标准对话框，将文件压缩为 zip、tar 和其他压缩格式：支持的格式列表取决于添加的压缩插件和外部压缩工具（请参阅 压缩处理）。

如果可用格式的列表太长而无法适应窗口，Double Commander 将把它们放入下拉菜单中（在这种情况下，请使用带有“=>”标签的复选框）。

配置 按钮的操作取决于所选的压缩工具：

  • 压缩插件 (WCX): 打开插件自己的设置窗口（如果选定的插件支持此功能）；

  • 外部压缩工具: 打开一个窗口，提供额外压缩参数的输入字段（当使用 %S 变量时）。

我们还可以通过 解压缩文件 （内部命令 cm_ExtractFiles）来解压文件。当您从压缩包中复制文件时，将调用相同的对话框。

在 解压缩与文件掩码匹配的文件 行中，您可以指定一个掩码，以确定将解压哪些文件。掩码的历史记录可以在下拉列表中找到。 下一行指定了解压到文件夹（压缩包将被解压到该文件夹）。可以手动更改或通过单击最右侧的浏览按钮进行更改。

将每个文件解压到单独的子文件夹（名称采用压缩文件文件名） – 如果选中，Double Commander 将在目标目录中创建一个与压缩文件同名的文件夹，并将文件提取到该文件夹中。您可以使用此选项一次提取多个压缩包，每个压缩包都将提取到目标文件夹中的其同名文件夹中。未确定的复选框状态（三态）启用智能提取：如果压缩包的根目录中有多个对象，则压缩包的内容将解压到与压缩包同名的文件夹中，但如果只有一个对象，则内容将原样解压。默认情况下，它未选中，并且其状态不会在下次调用提取命令时保存。

如果文件与压缩包一起存储，则解压缩路径名 – 如果选中，存档中的目录结构将被保留。否则，所有文件将被提取而不重新创建目录结构。

复选框 覆盖现有文件 禁用覆盖现有文件的确认。

测试压缩文件 （内部命令 cm_TestArchive）将验证所选压缩文件或光标下文件的内容完整性。如果压缩内容损坏，将报告错误消息。支持的格式列表取决于添加的压缩插件和外部压缩工具。

项目 分割文件 和 合并文件 （内部命令 cm_FileSpliter 和 cm_FileLinker）

将文件拆分到文件夹 – 指定拆分文件的目标文件夹。默认情况下，将使用非活动文件面板的目录。

下一组参数允许选择拆分部分的大小或数量。您可以输入自己的值或使用预设值之一：

  • 1457664 B - 3.5" – 3.5 英寸软盘;

  • 98078 KB - ZIP 100 MB – ZIP 驱动器;

  • 650 MB - CD 650 MB – CD 驱动器大小;

  • 700 MB - CD 700 MB – CD 驱动器大小;

  • 4482 MB - DVD+R – DVD 驱动器大小。

如果您使用 部分数量 参数，部分的大小始终以字节为单位显示。

如果启用 需要一个 CRC32 验证文件，Double Commander 将创建一个文本文件，文件名为源文件名，扩展名为 ".crc"，其中包含源文件的名称、大小和校验和（使用 CRC32 算法）。这是一种简单的方法，可以检查文件是否正确组合，以及在移动过程中其部分是否未更改。如果此文件不可用，Double Commander 将显示警告，但这不会阻止您获取原始文件（但是，在这种情况下，程序将无法添加原始文件扩展名）。

单击确定按钮后，您将在目标目录中获得多个文件。作为文件扩展名，Double Commander 使用三位或更多（如果需要）数字的顺序编号。要再次合并文件，请选择第一部分并单击 合并文件 （cm_FileLinker）。或者选择所有部分，您还可以指定合并的分割部分的顺序。

此功能非常有用，如果您有小驱动器和大文件。可以移动文件部分。请记住，此工具不会创建任何特定的容器或压缩包，而只是拆分合并文件，因此可以获取文件的部分并安全地通过其他程序合并在一起。

项目 计算校验码... （内部命令 cm_CheckSumCalc）用于计算 校验码 以检查文件完整性。以下校验码可用： BLAKE2 (BLAKE2s, BLAKE2sp, BLAKE2b, BLAKE2bp), BLAKE3, CRC32, HAVAL, MD4, MD5, RIPEMD128, RIPEMD160, SFV, SHA, SHA2 (digest sizes 224, 256, 384 or 512 bits), SHA3 (digest sizes 224, 256, 384 or 512 bits), Tiger/192.

保存校验码文件为： – 行指定计算的校验码的名称和路径。

为每个文件分别创建 MD5/SHA1 文件 – 如果对多个文件计算校验码，此选项允许为每个目标文件创建单独的校验码文件。

任务完成后打开校验码文件 – 任务完成后，校验码文件将在 文件查看器 中打开。

File format – 允许指定行结束类型和目录分隔符，而不是 Windows（CRLF 和 "\"）和类 Unix 系统（LF 和 "/"）中常用的值。

项目 验证校验码... （内部命令 cm_CheckSumVerify）将开始验证光标下的校验码文件中的校验码列表。您还可以通过双击校验码文件（或使用 Enter 键）来运行此命令。

Double Commander 可以逐个检查文件：选择文件，运行此命令，现在输入其校验码（或从剪贴板粘贴）并选择算法。

您可以使用其他程序生成的校验码文件，但首先确保 Double Commander 支持其算法和格式。

项目 擦除 （内部命令 cm_Wipe）用于安全删除（默认快捷键 Alt+Del）：在从硬盘删除之前，所选文件将被重命名为随机名称，其内容将被覆盖三次（零、一次和随机数据）。您可以在 配置 > 选项... > 文件操作 中设置擦除次数。目录和链接将在删除之前仅被重命名。

注意：固态硬盘的一些功能（垃圾回收、TRIM）显著增加了恢复已删除文件的难度或使其不可能，因此，在 SSD 上使用 cm_Wipe 命令可能是对设备资源的无谓浪费。

项目 删除 （内部命令 cm_Delete） – 将选定的文件或光标下的文件删除到回收站或永久删除。默认情况下，Double Commander 将文件移动到回收站，您可以在 配置 > 选项... > 文件操作 中更改此设置。删除到回收站可能并不适用于所有平台。

要删除存档中的文件（在这种情况下，文件将被直接删除），您需要添加合适的 WCX 插件或配置外部压缩程序，并且压缩包格式必须支持删除操作。

项目 退出 （内部命令 cm_Exit） – 关闭 Double Commander。

2.1.2. "标记"

此组中有一些命令可以以不同方式选择文件。

第一和第二项 选择一组文件 和 取消选择一组文件 （内部命令 cm_MarkPlus 和 cm_MarkMinus）。这些命令通过掩码选择文件组。使用此掩码，您可以指定文件名或扩展名（可以指定多个值，用分号 ";" 分隔，且不带空格）。此外，还有一个带有掩码历史记录的下拉列表。例如，选择所有具有 .txt 扩展名的文件：

定义... 按钮允许创建新的搜索模版。

如果 查找重复文件 结果在活动面板中打开，则这些命令将以特殊模式工作：

通过文件名/文件后缀选择： – 与常规组选择对话框中的工作方式相同。您还可以使用搜索模版（）。

通过文件名/文件后缀取消选择： – 在前一个选项之后应用，排除相应的文件。

在每组中至少保留一个文件未选中： – 如果您想删除重复文件，但在每组中保留至少一个文件，这是一个有用的选项。首先，Double Commander 将应用方法 1：修改日期（最新、最旧）、大小（最大、最小）或组中的位置（组中的第一个或最后一个文件）将被比较。如果没有文件符合条件（例如，所有文件具有相同的修改日期或相同的大小），则将应用方法 2：在这种情况下，将比较修改日期或大小。

应用 按钮允许立即查看结果，而无需关闭此对话窗口。

第三和第四项是 选择全部 （内部命令 cm_MarkMarkAll）和 全不选择 （内部命令 cm_MarkUnmarkAll）。它们可以快速选择或取消选择面板中的所有项目。

第五项是 全部不选 （内部命令 cm_MarkInvert）。它使当前选定的文件未选中，反之亦然。

第六和第七项是 选择所有扩展名相同的文件 （内部命令 cm_MarkCurrentExtension）和 取消选择所有扩展名相同的文件 （内部命令 cm_UnmarkCurrentExtension）。这些功能有助于选择和取消选择与光标下文件具有相同扩展名的文件。

第八和第九项是 保存选择 （内部命令 cm_SaveSelection）和 恢复选择 （内部命令 cm_RestoreSelection）。第一个命令将当前选定的项目保存在内部缓冲区中。然后，第二个命令可以用来重新选择之前选定的相同项目。

第十和第十一项是 保存选择到文件 （内部命令 cm_SaveSelectionToFile）和 从文件加载选择 （内部命令 cm_LoadSelectionFromFile）。与之前的命令类似，但使用纯文本文件（每行一个文件名）。

第十二项是 从剪贴板加载选择项 （内部命令 cm_LoadSelectionFromClip）。将根据剪贴板中的列表（每行一个文件名）选择文件和文件夹。

第十三和第十四项是 复制文件名到剪贴板 （内部命令 cm_CopyNamesToClip）和 复制包含完整路径的文件名 （内部命令 cm_CopyFullNamesToClip）。第一个命令将选定的文件列表复制到剪贴板，第二个命令也会复制，但带有完整的文件路径。

第十五项是 复制所有显示的列 （内部命令 cm_CopyFileDetailsToClip）。将复制选定项目的文件信息到剪贴板：每行一个文件，列的内容将以制表符分隔。

最后一项是 比较文件夹 （内部命令 cm_CompareDirectories）。将选择在活动面板和非活动面板中不在对方面板中的项目（默认情况下仅选择文件）。具有相同名称的文件将按修改日期进行比较，之后将选择较新的文件。

2.1.3. "命令"

第一个项目是 搜索 （内部命令 cm_Search）。此工具旨在根据指定的条件搜索文件和文件夹。还可以用于按文件内容搜索。有关详细说明，请参见 此页面。

新建搜索实例... （内部命令 cm_AddNewSearch） – 打开一个具有默认设置的搜索对话框的新实例。它还允许在长时间搜索进行时启动新的搜索实例以搜索其他内容。

查看当前搜索实例 （内部命令 cm_ViewSearches） – 将显示一个窗口，其中列出了搜索实例及其简要信息（状态、找到的文件数量、是否完成和文件掩码），并可以切换到其中一个实例。

第四项是 常用文件夹列表 （内部命令 cm_DirHotList）。此命令与按下 "*" 按钮相同。

选择此项，我们将获得一个弹出菜单，其中包含 常用文件夹列表，我们可以在其中选择：

  • 切换活动面板的当前路径到您创建的列表中的预配置文件夹；

  • 添加新书签：当前文件夹、选定文件夹（如有需要可选择多个）或光标下的文件夹；

  • 配置现有书签。

有关如何使用和配置它的详细信息，请参见专门的 常用文件夹列表 帮助页面。

同步文件夹... （内部命令 cm_SyncDirs）打开 文件夹同步工具：将扫描左侧和右侧面板的文件夹，您将看到它们内容的差异，并可以使它们相同。

运行终端 （内部命令 cm_RunTerm）在当前文件夹打开一个终端（设置在 工具 中）。

执行内部命令 （内部命令 cm_DoAnyCmCommand）打开一个窗口，列出所有内部命令及其描述。命令按类别分组，并提供快速过滤和排序功能：您可以执行所选命令、复制其名称或打开 内部命令 页面（某些命令支持参数）。

平面视图 （内部命令 cm_FlatView）将扫描活动面板当前文件夹中的所有子文件夹，并以一个列表显示所有文件。可以通过按 Esc 键中止扫描，再次调用该命令将返回正常模式。如果光标位于某个文件上，则在返回正常模式时，Double Commander 将把光标放在该文件上。

此外，还提供了 cm_FlatViewSel 命令，该命令也以一个列表显示所有文件，但仅适用于选定的文件和文件夹。再次调用此命令只会在基本文件夹中返回正常模式。

打开虚拟文件系统（VFS）列表 （内部命令 cm_OpenVirtualFileSystemList）将打开一个已添加的文件系统插件（WFX 插件）列表，这些插件使用自己的文件系统或提供对其他文件系统和设备（本地或远程）的访问。以及虚拟文件夹：

  • 此电脑 – 包含带有驱动器字母的正常磁盘和没有驱动器字母的设备（例如，通过 MTP 协议连接的 USB 设备，即智能手机、平板电脑或播放器）。仅限 Windows。

  • Linux – 列出您添加到 WSL 的所有 Linux 发行版及其根文件系统（Windows 10 及更高版本）。

  • 网络 – 访问共享网络资源（Windows）或使用 Gio 访问网络资源（Linux）。

  • 回收站 – 访问已删除到回收站的文件。

（这个列表是按字母顺序排列的，并且取决于系统和程序的语言设置。）

交换面板 （内部命令 cm_Exchange）交换左侧和右侧文件面板及所有打开的选项卡。

目标 = 来源 （内部命令 cm_TargetEqualSource）在非活动文件面板中打开相同的目录。

显示占用空间 计算并显示当前目录的子文件夹中所有文件的总大小。活动面板的大小列中的 <DIR> 文本将被总大小替换。

（返回顶部）

  • 在非活动面板中打开的连接；

  • 最后创建的连接。

映射网络驱动器... 和 断开网络驱动器连接... – 将调用 Windows 系统对话框以连接和断开网络驱动器。

使用网络路径复制名称 – 将复制网络上文件的完整名称到剪贴板：服务器名称、共享名称和相对文件路径。

2.1.5. "标签"

此组管理带标签的文件面板。

新建标签 （内部命令cm_NewTab） – 在活动面板上添加一个新标签（带有当前路径）并切换到它。

重命名标签 （内部命令cm_RenameTab） – 允许更改当前标签的标题。

在新标签中打开文件夹 （内部命令cm_OpenDirInNewTab） – 为光标下的目录打开一个新标签，但不切换到它（即在后台）。

关闭标签 （内部命令cm_CloseTab） – 关闭活动标签并将面板切换到下一个标签（如果关闭最后一个标签，则切换到上一个标签）。

关闭所有标签 （内部命令cm_CloseAllTabs） – 关闭所有非活动标签。如果它们包含锁定标签，将提示确认。

关闭重复标签 （内部命令cm_CloseDuplicateTabs） – Double Commander 将检查打开的标签列表并关闭重复标签（每个面板单独处理！）。使用以下优先级规则：

  • 锁定标签具有更高优先级；

  • 如果存在具有相同目录的锁定标签，则重命名的标签将被关闭；

  • 如果存在具有相同目录的重命名标签，但未重命名，则第二个标签将被关闭；

  • 锁定的重命名标签优先于未重命名的标签，因此第二个标签将被关闭；

  • 如果存在两个具有相同优先级的标签，则第一个标签将被保留（从左侧开始计数）。

标签选项 – 使用此子菜单可以更改标签的状态：

  • 正常 （内部命令cm_SetTabOptionNormal） – 将标签设置为正常：Double Commander 默认创建的普通标签。

  • 锁定 （内部命令cm_SetTabOptionPathLocked） – 将标签设置为锁定状态，尝试在此标签内更改到另一个目录将提示在新标签中打开。

  • 锁定，但允许更改文件夹 （内部命令cm_SetTabOptionPathResets） – 如果您以此方法锁定标签，则可以在此标签内更改到另一个目录，但在切换到另一个标签（或重新启动 Double Commander）后，标签将恢复为锁定标签的原始目录。

  • 锁定，并在新标签中打开文件夹 （内部命令cm_SetTabOptionDirsInNewTab） – 当您尝试进入面板内的任何目录时，该目录将以新标签的形式打开。

剩下的子菜单项将执行相同的操作，但适用于所有标签。

注意：默认情况下，Double Commander 不允许在没有确认的情况下关闭锁定标签（在文件夹标签设置部分中的确认关闭锁定标签）。

切换到下一标签 （内部命令cm_NextTab）和 切换到上一标签 （内部命令cm_PrevTab） – 分别切换到右侧（下一个）标签和左侧（上一个）标签。

保存标签到文件 （内部命令cm_SaveTabs）和 从文件加载标签 （内部命令cm_LoadTabs） – 当前标签的名称及其关联目录可以保存在 .tab 文件中，并从该文件加载。

将当前标签保存到新的收藏标签 （内部命令cm_SaveFavoriteTabs） – 将当前打开的标签保存到收藏标签集中。稍后，您可以使用下一个菜单项重新加载这些保存的设置。

从收藏夹标签加载标签 （内部命令cm_LoadFavoriteTabs） – 将显示一个弹出菜单，您可以选择之前保存的标签集之一。

最后两个项目用于快速跳转到文件夹标签和收藏夹标签配置部分。

（返回顶部）

2.8. 文件面板

文件面板显示当前目录中文件的名称。Double Commander 可以将文件面板垂直（左右，默认）或水平（上下）排列，使用 水平面板模式 项在 显示 菜单中或 Ctrl+Shift+H。

默认情况下，两个面板的宽度相同。大小以左（或上）面板的百分比设置，有几种方法可以更改它：只需用鼠标拖动面板分隔符（程序将在工具提示中显示左面板的宽度），使用右键菜单中的固定大小比例（20/80、30/70、40/60、50/50、60/40、70/30、80/20）或 cm_PanelsSplitterPerPos 命令。

显示文件面板内容的一般参数收集在 文件视图 和 文件视图扩展 配置部分。

提供几种文件列表视图模式。

详细信息列视图 – 每行一个文件，带有默认或用户定义的列。对于文件夹和链接，Double Commander 显示 <DIR> 和 <LNK>。选项可在 列 配置部分中找到，您可以在 自定义列 部分中添加或更改列集。列视图用于大多数插图。

列表视图 – 仅文件名称。如果可能，显示多个列。选项可在 文件视图 > 简要 配置部分中找到。

缩略图视图 – 文件以小图像形式显示。如果可能，显示多个列。您可以在 杂项 配置部分中更改缩略图的大小。

Double Commander 可以利用操作系统的功能来创建缩略图或自行创建缩略图。您可以选择允许将缩略图缓存保存到磁盘，或仅在程序关闭之前将其保留在内存中（请参阅 杂项）。

创建缩略图的方法和支持的文件格式取决于操作系统。

  • Windows: Double Commander 使用操作系统的功能（如 Windows 资源管理器），WLX 插件支持缩略图创建，并将列表补充为 内部查看器 支持的图像格式。

  • macOS: Double Commander 使用操作系统的功能（Quick Look）并将列表补充为 内部查看器 支持的图像格式。该程序还支持 OpenDocument 格式（.odt、.ods、.odp、.odg），并将在使用 Quick Look API 之前尝试创建缩略图（如果文件包含 "Thumbnails/thumbnail.png"）。

  • Linux 和其他类Unix系统: Double Commander 为以下格式创建缩略图:

    • 由 内部查看器 支持的图像格式;

    • XCF 图像格式（如果可用 ImageMagick 库）;

    • 一些流行的视频格式（如果可用 FFmpegthumbnailer 库）：.avi、.flv、.mkv、.mov、.mp4、.mpeg、.mpg、.vob、.webm、.wmv;

    • MP3 文件的嵌入封面艺术;

    • OpenDocument 格式（如果文件包含 "Thumbnails/thumbnail.png"）：.odt、.ods、.odp、.odg。

Double Commander 还会为文本文件（纯文本和其他非二进制文件）创建缩略图。这些文件的缩略图不会保存到磁盘。

Double Commander 可以在文件名称左侧显示文件图标（默认启用），以显示文件类型和与程序的关联（系统或 内部）。DC 使用其自己的图标主题和/或根据系统设置的图标（请参阅 图标 设置部分）。

该程序还允许指定文件类型，其名称应为不同颜色，请参阅 颜色 > 文件类型 设置部分以获取更多详细信息。

当前文件将被细矩形包围，带或不带颜色填充，所选文件的名称将显示为红色。您可以在 颜色 > 文件面板 设置部分中更改使用的颜色和光标的外观。

Double Commander 可以在鼠标光标悬停在文件上时显示工具提示，请参阅 工具提示 设置部分以获取详细信息。


选择文件

有几种选择文件的方法。

要选择光标下的文件或目录，只需按 Insert 键（选择后，Double Commander 将自动将光标移动到下一个文件）或 空格 键。如果使用 空格 键选择目录，程序将计算该目录的总大小并在相应列中显示。

您可以 选择 左键或右键来选择文件：

  • 左键 – 鼠标的工作方式与 Windows 资源管理器相同，右键仅用于打开上下文菜单。在此模式下，您可以通过单击任何未选择文件的名称或 ".." 项来取消选择所有选定的文件。您还可以使用适当的内部命令（见下文）。

  • 右键 – 左键的工作方式相同，右键允许通过单击文件名来选择文件。稍微长时间按住右键将调用文件的上下文菜单。要选择多个连续文件，您可以按住右键并移动鼠标光标在文件列表上。要取消选择所有选定的文件，您可以使用适当的内部命令（见下文）。

您可以使用带有修饰键的鼠标单击：Ctrl（选择下一个文件，列表中的任何位置）和 Shift（将多个连续文件添加到选择中）。要选择多个连续文件，您可以按住 Shift 并使用上下键。

还可以通过单击文件图标来选择文件，您可以在 鼠标 设置部分启用此功能。同时，Double Commander 将把光标移动到所选文件上，就像使用带有 Ctrl 键的鼠标单击一样。在缩略图视图中，当您单击图像的左侧部分（1/4 或 25%）时，这种方式有效，因此您可以通过单击右侧图像部分来设置光标而不进行选择。此功能方便用于取消选择大量文件：单击任何选定文件的图标，然后单击其名称。

您可以完全禁用鼠标选择：左键将仅设置光标，右键将调用上下文菜单。

可用特殊 命令 选择和取消选择文件，这些命令允许使用不同条件：按名称或名称的一部分、文件类型、属性等。其中一些已添加到 标记 菜单中，您可以 分配热键 或将按钮添加到 工具栏。或者打开一个窗口，列出所有内部命令及其描述（内部命令 cm_DoAnyCmCommand，默认 Shift+F12）并转到 标记 类别。


拖放

Double Commander 支持使用鼠标拖放文件（即按住左键）到外部应用程序的窗口（例如文本或图形编辑器或查看器）、工具栏以及在目录之间复制/移动文件。如果该操作是可能的，"+" 符号将显示在鼠标光标附近。

将文件拖到工具栏的空白处时，将创建一个带有外部命令的按钮，传递的文件将用作命令。将文件拖到按钮上仅在该按钮是带有外部命令的按钮时才可能：该命令将启动，文件将作为启动参数传递给它（请记住，在这种情况下按钮命令的参数将被忽略）。

对于文件复制或移动（使用 Ctrl 或 Shift 键）操作，可能的目标包括：相对文件面板、活动或非活动面板中的任何文件夹（文件夹名称将用框突出显示）、任何打开选项卡的目录（只需将文件拖到所需选项卡的标题上）。使用 ".." 将文件复制到父目录。如果支持添加文件，Double Commander 还可以将文件复制到压缩包中。您可以 启用或禁用确认对话框 并选择默认操作（请参阅 <DefaultDropEffect> 的描述）。

使用右键拖动将显示以下操作的菜单：复制、移动、创建链接（硬链接）、创建符号链接 和 取消。如果选择了多个文件，Double Commander 将仅为列表中的第一个文件提供创建链接的选项。

您可以使用 Ctrl+Shift 创建符号链接。

当您将文件拖到另一个应用程序的窗口时，Double Commander 使用系统实现的拖放功能。使用 Alt 键，您可以调用替代操作（例如，当将文件拖到 Windows 资源管理器窗口时，将创建一个快捷方式，而不是复制文件）。


上下文菜单

文件的上下文菜单（右键菜单）的外观和内容取决于操作系统。在 Windows 中，Double Commander 使用系统上下文菜单（如 Windows 资源管理器）和“操作”子菜单（见下文），在其他操作系统中，Double Commander 创建自己的上下文菜单，包含典型操作。如果您更喜欢使用右键选择模式，则需要稍微长时间按住右键才能调用此菜单。您可以使用内部命令 cm_ContextMenu 来调用上下文菜单（默认 Shift+F10）。

Double Commander 在 Linux 和其他类 Unix 系统中的上下文菜单：

  • 打开 – 在默认程序中打开光标下的文件（将使用系统文件关联）。

  • 操作 子菜单 – 包含所有添加到光标下文件扩展名的 文件关联 的操作。您还可以添加一些 附加操作。

  • 使用打开 子菜单 – 包含适合打开文件的应用程序列表。要在 macOS 中获取该列表，Double Commander 使用 Launch Services API，在其他系统中，该程序遵循 桌面菜单 规范。
    附加项 其他...

    • macOS: 将会打开所有已安装应用程序的列表。

    • Linux 和其他类 Unix 系统: 将打开应用程序选择窗口，您可以从所有可用应用程序的列表中选择一个应用程序或手动指定命令行以打开文件。在这里，您还可以为文件类型保存额外的系统关联，添加新的关联或更改默认应用程序（Double Commander 将使用 ~/.config/mimeapps.list 文件（请参阅 MIME 类型与应用程序之间的关联）。

  • 服务 – 仅 macOS，类似于“服务”系统子菜单。

  • 分享... – 仅 macOS，类似于“分享”系统子菜单。

  • 在此打包... – 调用 命令打包选定的文件，当前目录将用作目标目录。

  • 在此解压缩... – 调用 命令解压缩选定的文件，当前目录将用作目标目录。仅当在压缩插件或外部压缩程序的设置中指定了文件扩展名时，此项才会添加。

  • 调用基本文件操作: 移动 (F6), 复制 (F5), 删除 (F8), 重命名 (F2).

  • 使用剪贴板复制和移动文件的命令: 剪切, 复制, 粘贴

  • 显示文件属性 – 显示 文件属性窗口。

在空白区域单击鼠标右键，在 Linux 和其他类 Unix 系统的文件面板中会调用一个菜单，包含以下项目：

  • 刷新 – 重新读取面板中目录内容的命令。

  • 排序方式 子菜单 – 包含切换排序方式的命令：按名称、文件扩展名、大小、修改日期或属性。还添加了 反向排序 命令，可以切换排序方向。

  • 粘贴 – 用于使用剪贴板复制和移动文件。

  • 新建 子菜单 – 包含创建目录（F7）和文本文件（Shift+F4）的命令。此子菜单还包含新文件的模版，如果存在且不为空的系统模版目录（在 macOS 中不可用）。

  • 显示文件属性 – 显示 文件属性窗口。

在打开的压缩包文件或文件系统插件（WFX）中，右键菜单始终相同，并且不依赖于操作系统：

  • 打开 – 在默认程序中打开文件，就像使用双击或按 Enter 一样（将使用 内部 和系统文件关联）。默认情况下，在压缩包文件内部，此命令将显示已打包文件的属性窗口（请参阅 压缩包处理），您可以选择在关联程序中打开文件，但 Double Commander 允许直接配置在关联程序中自动打开文件（使用 <AutoExtractOpenMask>）。

  • 查看 – 在查看器中打开文件（F3）。

  • 调用多个文件操作: 复制 (F5), 重命名 (F2) 和 删除 (F8).

  • 显示文件属性 – 显示文件属性窗口。内容取决于虚拟文件系统的功能。在压缩包内部，这将是已打包文件的属性窗口（请参阅 压缩包处理）。

请记住，并非所有文件操作在每种情况下都能正常工作。例如，如果某些压缩包格式不支持删除文件， Double Commander 将隐藏 删除 菜单项。


重命名文件

Double Commander 允许直接在文件面板中重命名文件。有几种方法可以调用此操作：内部命令 cm_RenameOnly（默认情况下为 F2 或 Shift+F6）、左键单击 或右键菜单中的相应项。

默认情况下，Double Commander 将完全选择文件名，但您可以启用 不带扩展名的文件名选择。此外，重复按下热键可以按部分选择文件名，以下字符用作分隔符：空格、连字符、下划线和句点。文件夹名称始终完全选择。

要确认名称更改，请使用 Enter 键或编辑字段右侧的重命名确认按钮。要取消未确认的更改（退出编辑模式），请使用 Esc 键或在编辑字段外单击鼠标。您可以在编辑字段外单击鼠标以确认重命名（如在 Windows 资源管理器中），请参阅<RenameConfirmMouse>的描述。

对于批量重命名文件，可以使用适当的工具。


快速搜索/过滤

使用快速搜索/过滤工具时，其面板显示在文件面板的底部。快速搜索用于搜索文件面板的当前内容：光标将移动到最近的匹配文件，当使用 上箭头 和 下箭头 键时，程序将仅在文件名匹配条件的文件上循环移动光标。快速过滤将隐藏所有不匹配条件的文件名（要重置过滤器，请使用 Esc 键或“X”按钮）。

  • 过滤器 – 搜索/过滤开关;

  • { – 名称必须以输入的第一个字符开头;

  • } – 输入的点“.”之前的最后一个字符必须匹配;

  • Aa – 区分大小写;

  • F – 过滤文件;

  • D – 过滤目录;

  • X – 禁用快速搜索并关闭面板。

Double Commander 支持拼音（将使用程序目录中的 pinyin.tbl 文件）。

您可以在 按键 设置部分中设置调用快速搜索/过滤面板的方式，或使用 cm_QuickSearch 和 cm_QuickFilter 命令。

在快速搜索面板中更改的所有参数将在下次快速搜索/过滤调用时保留（在关闭程序之前），您可以在 快速搜索/过滤 设置部分中更改它和默认值。

2.9. 状态栏

显示有关文件面板内容的一些详细信息。

如果设置了列表视图，则状态栏将显示所选文件的大小及其数量，以及面板中所有文件的总数量和大小。

如果设置了列表视图或缩略图视图，则此处将显示光标下的文件名、大小、修改日期和属性。如果选择了文件，则状态栏将显示与列表视图相同的信息。

如果启用了扁平视图（没有子目录），则光标下的文件名将以相对路径显示。如果选择了文件，则状态栏将显示与列表视图相同的信息。

2.10. 命令行

用于手动输入命令，您可以快速运行或打开当前目录或 PATH 中的文件（如有必要，可以添加其他参数）。

活动目录的路径显示在行首（在屏幕截图中为 C:\Program Files\Double Commander\）。右侧带箭头的按钮打开已输入命令的历史记录。使用 Shift+Del 删除历史记录中不必要的条目。

有几个 内部命令 可用于处理命令行，还可以查看 按键 设置部分中的输入选项。

如果在 设置 中隐藏了命令行，您仍然可以使用它。在这种情况下，命令行将在命令运行后自动隐藏。您可以使用 Esc 键（如果输入字段为空）或将光标移动到文件面板（或以其他方式切换焦点）来隐藏它。

支持环境变量，在 Linux 和其他类 Unix 操作系统中，您还可以使用波浪号符号“~”来表示当前用户的主目录。

Double Commander 独立处理 cd 命令，您可以使用它来更改活动面板中的当前目录（仅适用于本地文件系统或作为本地磁盘挂载的远程资源）。如果您使用带有文件名的路径，Double Commander 将更改目录并将光标放在该文件上。如果您仅指定文件名，Double Commander 将在当前目录中将光标放在该文件上。可以使用 cd .. 命令导航到父目录，运行不带参数的命令将打开当前用户的主目录。

您可以使用斜杠：cd \ 进入当前驱动器的根目录（Windows），cd / 进入文件系统的根目录（Linux 和其他类 Unix 系统）。

在 Windows 中，您可以通过简单地指定驱动器字母和冒号来更改驱动器：c:、d: 等。

右键单击将显示常用输入框菜单，其中包含标准命令 剪切、复制、粘贴、删除、全选 和子菜单 插入 Unicode 控制字符。
