1. 压缩程序 插件 (WCX)


Base64

Base64 插件提供按照 Base64 方案对文件进行编码和解码的功能，Base64 是用可打印 ASCII 字符对二进制数据进行编码的标准之一。


CPIO

用于解包 CPIO 归档的打包插件。


DEB

用于解包 dpkg 包管理系统的包文件（Debian 及基于其的发行版）的打包插件。

适用于 Linux 及其他类 Unix 操作系统（在 Windows 上，Double Commander 使用 SevenZip 插件处理该格式）。


RPM

用于解包 Red Hat 包管理系统的包文件（Red Hat、SUSE 等发行版）的打包插件。


SevenZip

用于处理 .7z、.arj、.cab、.cpio、.cramfs、.deb、.dmg、.fat、.hfs、.iso、.lha、.lzh、.ntfs、.squashfs、.taz、.vhd、.wim、.xar 和 .z 等文件的打包插件。也可添加 7-Zip 支持的其他文件类型。

注意：部分格式为只读，意味着只能查看内容、解包和测试。

仅限 Windows。


UnRAR

用于解包 RAR 归档的打包插件。

要使用该插件需要 UnRAR 库，插件将使用 unrar.dll（Windows）、libunrar.dylib（macOS）、libunrar.so 或 libunrar.so.5（Linux/类 Unix）。

Windows 的 Double Commander 分发包和 Linux 的便携版通常包含所需库；在其他情况下，可通过操作系统的包管理器安装（例如 Debian/Ubuntu 的 libunrar5 或 Arch 的 libunrar）。也可以在 此处 查找合适的库文件（需将文件复制到系统目录或 Double Commander 可执行文件所在目录）。

该插件还支持将文件打包为 RAR，需在插件设置中指定 WinRAR.exe 的路径（仅限 Windows）或命令行工具 rar。


Zip

用于处理 .bz2、.gz、.jar、.lzma、.tar、.tbz、.tgz、.tlz、.txz、.tzst、.xz、.zip、.zipx 和 .zst 等归档的打包插件。该插件也可以打开基于 ZIP 的文件（如 EPUB、Office Open XML、OpenDocument、CRX、XPI 等）。

需要额外的压缩库：

  • bzip2: bz2.dll（Windows）、libbz2.dylib（macOS）、libbz2.so.1（Linux/类 Unix）。

  • XZ: liblzma.dll（Windows）、liblzma.dylib（macOS）、liblzma.so.5（Linux/类 Unix）。

  • Zstandard: libzstd.dll（Windows）、libzstd.dylib（macOS）、libzstd.so.1（Linux/类 Unix）。

Double Commander 的 Windows 分发包已包含这些库，且在常见的 Linux 桌面发行版上这些库通常也会默认安装。

2. 内容 插件 (WDX)


内置 Exif 插件

程序包含一个内置的 Exif 内容插件，用于读取 JPEG 文件元数据中的部分信息：日期、像素尺寸、图像方向和相机信息等。

仅限 JPEG 图像。


AudioInfo

该内容插件显示音频文件的信息，包括技术信息（时长、比特率、声道数等）和元数据（标题、艺术家等）。

支持的元数据类型：ID3v1（1.0、1.1）、ID3v2（2.2、2.3、2.4）、APE（1.0、2.0）、Vorbis 注释。


DEB

该内容插件显示 dpkg 包管理系统（Debian 及其衍生发行版）包文件的信息：包名、版本、描述、依赖关系等。


RPM

该内容插件显示 Red Hat 包管理系统（Red Hat、SUSE 等）包文件的信息：包名、版本、描述等。


Lua 脚本

plugins/wdx/scripts 目录包含若干以 Lua 编写的内容插件示例（详情见 Lua 脚本 帮助页）。

这些脚本不会自动添加到程序设置中。

  • descriptionwdx.lua – 从 descript.ion 读取文件描述。

  • fulltextodtwdx.lua – 在 OpenDocument 文本（ODT）文件中查找文本。需要 odt2txt。

  • simplewdx.lua – 含若干字段的简单示例。

  • textlinewdx.lua – 依次返回纯文本文件（.txt、.ini）的前五行中的各行。

3. 文件系统 插件 (WFX)


FTP

此文件系统插件用于处理以下网络协议：FTP、FTPS、FTPES、SFTP 与 SSH+SCP。插件还支持通过代理服务器（HTTP、SOCKS4 或 SOCKS5）工作。

需要额外的库：

1) SSL 与 TLS 协议

加密库 OpenSSL（版本 3.1, 3.0, 1.1.1, 1.1.0, 1.0.2, 1.0.1）：

  • Windows：插件包含常见文件名列表，所需文件可被自动定位。
  • macOS：libssl.N.dylib 和 libcrypto.N.dylib（其中 "N" 为版本号），或 libssl.dylib 与 libcrypto.dylib。
  • Linux/类 Unix：libssl.so.N 与 libcrypto.so.N（其中 "N" 为版本号）。

Windows 下的 OpenSSL 二进制可在 此处 找到；其他平台一般通过系统包管理器安装（例如 Debian/Ubuntu 的 libssl3 或 libssl1.1，Arch 的 openssl，macOS 的 Homebrew 包 openssl）。

注意：在 Windows 上，库文件应位于 Double Commander 可执行文件附近或 Windows 系统目录中。

可选地，在 Linux 上插件也可以使用 GnuTLS（libgnutls.so.N）库。

2) SSH 协议

需要 libssh2 库：插件将使用 libssh2.dll（Windows）、libssh2.dylib（macOS）或 libssh2.so.1（Linux/类 Unix）。

Windows 的 Double Commander 分发包包含该库；其他平台可通过包管理器安装（例如 Debian/Ubuntu 的 libssh2-1、Arch 的 libssh2，macOS 的 Homebrew 包 libssh2）。

插件支持使用 SSH 认证代理（例如 OpenSSH 的 ssh-agent、PuTTY 的 Pageant）。


MacCloud（Alpha 版本）

macOS 官方云存储插件，支持：

  • Dropbox
  • Yandex
  • OneDrive
  • Box
  • Amazon S3
  • BackBlaze B2
  • Alibaba Cloud OSS
  • Tencent Cloud COS
  • Huawei Cloud OBS
  • Qiniu KODO
  • Upyun USS
  • S3 Compatible

仅限 macOS。

4. 查看器 插件 (WLX)


MacPreview

基于 Quick Look 的通用预览插件（支持办公文档、图片、视频与音频等），类似 Finder 中的预览。

仅限 macOS。


Explorer 预览

用于显示丰富预览的插件。IPreviewHandler 接口支持多种格式（取决于操作系统版本及已安装的应用），默认情况下检测字符串仅包含 HTML 文件。

注意：预览处理程序可能无法显示文件中包含的所有信息。

仅限 Windows。


富文本格式 (RTF)

用于查看 RTF 文件的插件。

仅限 Windows。


WlxMplayer

用于预览视频文件的插件。需要 mplayer。

仅限 Linux。


Windows Media Player

用于播放音频和视频的插件。该插件使用 IWMPPlayer4 接口（ActiveX），需要 Windows Media Player 9 或更高版本。

仅限 Windows。

5. 搜索 插件 (DSX)


DSXLocate

该插件使用 locate 及其文件数据库进行搜索。可按文件名、部分名称或通配符掩码搜索（符号 "*" 表示匹配任意数量的字符）。

适用于 Linux 及其他类 Unix 操作系统。
